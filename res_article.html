res.send vs. res.sendFile vs. res.render vs. res.json

All of them are responsible for sending a server response. Now the question is: when to use what? Read our thoughts in this article.

Jane Seymour A few hours ago · 5 min. read
Share this

res.send

This is probably the most dubious option of all. Because as a beginner when we start learning the server basics - probably the stage where we are not even aware of how dynamic data gets served - most of us tend to assume that the server is there only to serve up specific files in specific routes.

While there is nothing wrong with this thought, the send() function of the res object functions exactly opposite to this beginner/learner's mindset. It's not there to serve up files. Nor its job is to send JSON data (which is the result of making an API call from the front end).

So what exactly is its job? Merely send a raw response across to test whether our app is working or not? We are not sure. And it is, for this reason, we don't prefer using it. Please let us know your thoughts on this.

Usage example: res.send(‘hello world’)

res.sendFile

Here’s a server response we love a lot. It’s pretty simple and self-explanatory. We have a sendFile() function (of the res object) whose job is to send across a file when the user hits or accesses a specific endpoint. Makes sense. And at the same time clearly explains to you the basic purpose of a server – which is to serve files.

Usage example: res.sendFile(‘index.html’)

res.render

Another server response that makes sense. Especially useful if you want to perform CRUD operations without making API calls (since this might seem like overkill – especially for a basic operation like CRUD). The only problem you might face is that you have to learn views (EJS, PUG, etc) to make use of the render function, since it does not work with HTML.

Moreover, as mentioned earlier, you can easily send/pass dynamic data using render function easily.

Usage example: res.render(‘index.ejs’)

res.json

This is the best server response method you can opt for if you decide to make API calls from your front end to get a hold of the server response. More specifically, res.json is especially useful for sending across dynamic data – whether you are using an exclusive framework (like React) or simple HTML as your front end.

Usage example: res.json({"msg": "data fetched sucessfully"})

Our Verdict

Never use res.send (unless you want to simply test out the server by printing out ‘hello world’). Use sendFile when you want to have a no-frills, simple HTML as your frontend. Use render if you want to get rid of API calls. Use res.json if you want to use API calls to serve up dynamic data.
